Debug and Trace playground
Compare the Erigon and Geth clients

This app allows to call the trace_transaction, or debug_traceTransaction method.


trace_transaction

trace_transaction is one of the methods available in Erigon to trace, and it takes a transaction hash as input, allowing you to see the internal functions calls made into a smart contract.

This GitHub Gist has a detailed explanation and a transaction analysis example.

trace_transaction guide

debug_traceTransaction

The debug function in Erigon is very useful during the development of a smart contract to understand why a transaction is failing.

debug_traceTransaction takes a transaction hash as input and returns logs of low-level opcode that can be used to show what happens step by step during the process and what is the reason for it failing.

This GitHub Gist has a detailed explanation and a transaction analysis example.

Instructions
  1. Have access to an Erigon and Geth node endpoints
      2. You can deploy one with Chainstack!
      Sign up on Chainstack.
      Deploy a node.
  2. Insert the enpoints URL in the node input fields.
  3. Paste a transaction hash in the transaction field.
  4. Choose if you want to trace or debug the transaction.
  5. Click "Compare" if you only want to see the statistics (This will not display the response).
      6. Note that trace only works with Erigon.
      The analysis section with display the time needed to retrieve the data, as well as the size.
      The last section will display the JSON response.
